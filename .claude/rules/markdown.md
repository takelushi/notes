# Markdown ファイル規約

Markdown ファイルには以下の規約を守ってください。

## 言語

- すべて日本語・敬体で記載してください。
- NG: 英語
- NG: 日本語の常体・口語

**例外**:

以下は英語で記載してください。エージェントの精度向上のためです。

- `CLAUDE.md`
- `.claude/skills/**/SKILL.md` のヘッダー（name, description など）

## ファイル種類

| 種類             | 保存場所    | 説明                                     |
| ---------------- | ----------- | ---------------------------------------- |
| README           | `README.md` | プロジェクト全体の概要                   |
| CLAUDE.md        | `CLAUDE.md` | Claude Code のプロジェクトメモリ         |
| Claude Code 設定 | `.claude/`  | Claude Code の動作に影響する設定ファイル |
| ナレッジノート   | `**/*.md`   | 個人的なナレッジベースノート             |

## 形式

### 基本

- 見出し 1 (`# `) は最初に一度だけ使用してください。
- 見出しは階層的に使用してください。見出しレベルを飛ばさないでください。
- Markdown の記法を適切に使用してください。
- 箇条書きリストは `- ` を使用してください。 `* ` は使用しないでください。
- コードブロックはトリプルバッククォート (```) および言語指定を使用してください。

### ファイルの参照

- ファイルの参照には相対パスを使用してください。
- ファイル指定 `[タイトル](../README.md)`
- セクション指定 `[タイトル](../README.md#セクション名)`
- 行指定 `[タイトル](../README.md#L10)`

## 禁止事項

- 一時的な情報を記載しないでください。
- 機密情報を記載しないでください。
